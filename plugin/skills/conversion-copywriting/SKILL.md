# Conversion Copywriting

Smallest set of sentences that move reader from "I'm here" to "I'll do the thing." Every word competes for the click.

## When to use

- Landing page (product, feature, campaign)
- Email (sales sequence, lifecycle, re-engagement, launch)
- Ad copy (search, social, display)
- Blog post driving downstream action
- Sales / pricing / comparison page
- Copy with measurable success metric beyond reads

Skip: internal docs, neutral reference, technical writing where persuasion is off-mission.

## Five questions before writing a word

1. **Who is the reader?** Specific — "ops manager at 50-person SaaS asked to cut tooling cost 20%"
2. **What do they believe now?** Their current story about themselves and the problem
3. **What do we want them to do?** ONE action
4. **What's stopping them?** Objections, doubts, alternatives
5. **What's the proof?** Testimonial, data, demo, guarantee

Can't answer all five → research is the bottleneck, not copy.

## Frameworks (use one)

| Framework | Best for |
|-----------|----------|
| **PAS** (Problem/Agitate/Solution) | Pain-driven decisions, reader already feels problem |
| **AIDA** (Attention/Interest/Desire/Action) | Cold traffic |
| **Before/After/Bridge** | Transformation stories |
| **FAB** (Features/Advantages/Benefits) | Feature pages — most copy stops at features; conversion ends at benefits |
| **JTBD** (Job to be done) | Product replaces a different solution |

## Anatomy of a converting page

| Section | Job |
|---------|-----|
| Hero headline | Earn next 5 seconds |
| Sub-headline | Specify who and what |
| Lead visual | Show outcome, not interface |
| Primary CTA (above fold) | Make action obvious |
| Proof block | Names, logos, numbers — fast |
| Problem articulation | Reader thinks "yes, that's me" |
| Solution | 3 benefits, ordered by importance |
| Objection handlers | "But what about..." section |
| Secondary CTA | For readers who needed more |
| Risk reversal | Money-back, free trial, no card |
| Final CTA | "I'm ready" close |

Every page should know which sections it deliberately skips.

## Headlines — 80% of the work

Patterns that work:
- **Specific outcome + timeframe** — "Cut your CI bill 40% in one afternoon"
- **Named pain + named relief** — "Stop chasing payments. Get paid on time."
- **Before/after** — "From spreadsheet chaos to one source of truth"
- **Curiosity gap with payoff** — "The mistake every Postgres team makes (and the one-line fix)"
- **Direct address + stakes** — "Your onboarding is leaking money. Here's the fix."

Patterns that fail:
- Clever wordplay
- "Welcome to [Brand]" — wastes top line
- Vague benefit ("Better collaboration")

## Voice rules

- **Specific beats clever.** "Save $1,200/month" > "Big savings"
- **You / your beats we / our.**
- **Concrete beats abstract.** "300 invoices in 4 minutes" > "fast invoicing"
- **Active beats passive.**
- **Read out loud.** Sounds like press release → rewrite.
- **Cut every word that doesn't earn its space.**

## Calls to action

- One primary action per page
- Action verb + outcome: "Start free" > "Submit", "Get my plan" > "Click here"
- "Free, no credit card" by the button defangs hesitation
- Repeat CTA — readers scroll, scan, leave, return

## Common mistakes

- Writing about product, not reader
- Listing features as if they sell themselves
- Hedging ("We aim to help you potentially save...")
- One CTA at bottom of long page
- Stock photos of generic people laughing at laptops
- Headline requiring sub-headline to explain product
- Stacked testimonials without specifics
- "Game-changing", "revolutionary", "synergy" — banned

Emails specifically:
- Subject line that works as headline ≠ subject that gets opened
- One email doing 3 jobs (announce + educate + sell)
- Hero image instead of sentence (clients block images)
- Long form when 3-line nudge would convert better
- No P.S. — second-most-read line in any email

## Quick reference — diagnostic

| Symptom | Likely cause |
|---------|--------------|
| High traffic, low click | Headline/hero mismatch with source |
| High click, low signup | Friction in form/onboarding |
| Reads, doesn't act | Missing urgency, risk reversal, vague CTA |
| Bouncing fast | Lead doesn't match ad promise |
| "Looks great but I don't get it" | Buried what-it-is, over-rotated on benefits |

## Pre-publish checklist

- [ ] One reader, one action, named
- [ ] Headline survives 5-second test
- [ ] 3 benefits, ordered, each with proof
- [ ] CTA repeated, unambiguous
- [ ] Specifics replace adjectives
- [ ] Objections addressed before raised
- [ ] Read out loud — sounds human
- [ ] Mobile preview works

## Common Rationalizations

| Excuse | Reality |
|--------|---------|
| "I'll write 'professionally' and edit later" | 'Professional' = jargon-heavy + lifeless. Edit toward specificity from draft 1. |
| "Simple change" | 41% of agentic-LLM failures hide in trivial diffs (DAPLab). |
| "Time pressure" | Tech debt compounds. |

Default: run anyway.
